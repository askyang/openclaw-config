# 天气助手Agent部署指南

## 概述
这是一个定时执行的天气助手Agent，每天北京时间8:00自动获取北京天气信息并发送到指定的飞书群聊。

## 文件说明

### 1. 主要脚本
- `weather_assistant.sh` - 天气获取和消息生成脚本
- 功能：调用wttr.in API获取北京天气，格式化消息内容
- 执行权限：已设置为可执行

### 2. 定时任务配置
- `weather_cron.json` - OpenClaw定时任务配置
- 计划：每天8:00执行（北京时间）
- 目标：飞书群聊 `oc_cf380451c6f74d1584a778544225c1ff`

## 部署步骤

### 步骤1：测试脚本
```bash
bash /Users/itxueba/.openclaw/workspace/weather_assistant.sh
```

### 步骤2：配置定时任务
根据OpenClaw的定时任务系统，需要将`weather_cron.json`配置添加到定时任务中。

### 步骤3：验证飞书群聊ID
确保群聊ID `oc_cf380451c6f74d1584a778544225c1ff` 存在且可访问。

## 消息格式示例
```
🌤️ 早安！北京天气播报 (2026年03月31日)

北京: ☀️ +17°C (体感 +17°C), ↑12km/h 风速, 39% 湿度

⏰ 播报时间: 2026-03-31 19:43:38

💡 温馨提示：
• 记得根据天气调整着装
• 出门前查看实时天气
• 注意保暖防寒

#天气 #北京 #每日播报 #早安
```

## 故障处理

### 1. 天气API失败
- 脚本有备用方案，会显示默认天气信息
- 错误信息会被捕获，不会导致脚本崩溃

### 2. 飞书发送失败
- 需要确保OpenClaw有发送消息到该群聊的权限
- 检查群聊ID是否正确

### 3. 定时任务不执行
- 检查OpenClaw定时任务服务是否运行
- 验证时区设置（Asia/Shanghai）
