# 金字塔原理实战指南

## 核心原则

### 结论先行
最重要的一句话放在最前面。

**❌ 错误示例**：
> "最近几个月，我测试了很多AI编程工具。有Copilot、Cursor、Codeium、Claude Code等等。测试了之后发现，Claude Code最好用。"

**✅ 正确示例**：
> "Claude Code是目前最好用的AI编程工具。我测了Copilot、Cursor、Codeium等5个工具，它胜出。"

### 纵向关系
上层是结论，下层是依据。

**示例**：
```
结论：Claude Code最好用
├─ 理由1：速度快（比Copilot快2倍）
├─ 理由2：理解准（复杂需求不跑偏）
└─ 理由3：上下文长（能记整段代码）
```

### 横向关系
同一层的要点属于同一逻辑范畴。

**❌ 错误示例**（混杂）：
```
Claude Code的优势：
├─ 速度快
├─ 界面好看
└─ 我喜欢
```

**✅ 正确示例**（同类）：
```
Claude Code的三大优势：
├─ 性能优势：速度快、准度高
├─ 功能优势：上下文长、工具多
└─ 体验优势：界面简洁、操作流畅
```

## SCQA故事框架

### 结构说明
- **S (Situation)**：背景 - 大家都熟悉的情境
- **C (Complication)**：冲突 - 出现了什么问题/变化
- **Q (Question)**：疑问 - 怎么办？
- **A (Answer)**：方案 - 给出答案

### 实战案例

**案例1：工具推荐**

```
S (背景)：我在用AI写代码，试过很多工具
C (冲突)：但总感觉不够快，经常理解错我的意思
Q (疑问)：有没有更好的选择？
A (方案)：最近发现Claude Code，确实不一样
```

**展开后的文章开头**：
> "我在用AI写代码，试过很多工具。但总感觉不够快，经常理解错我的意思。最近发现Claude Code，确实不一样。"

**案例2：方法分享**

```
S (背景)：大家都想提高写作效率
C (冲突)：但AI写出来的东西，AI味太重，还要花时间改
Q (疑问)：怎么让AI写出人话？
A (方案)：我总结了一套"降AI味三步法"，今天分享给你
```

## DaLin 的爆款结构

### 公众号长文结构（2000字）

```
1. 开场（200-300字）
   ├─ 现象观察（大家都遇到的问题）
   ├─ 反转（但真相是...）
   └─ 点题（今天我要告诉你...）

2. 核心观点（1500字）
   ├─ 观点1（300字）+ 案例
   ├─ 观点2（300字）+ 案例
   ├─ 观点3（300字）+ 案例
   └─ 观点4（300字）+ 实战方法

3. 结尾（150字）
   ├─ 总结
   ├─ 行动号召
   └─ 固定签名
```

### 小红书短文结构（800字）

```
1. 钩子（50字）
   └─ 制造反差 / 晒结果

2. 干货（600字）
   ├─ 方法1 + 截图
   ├─ 方法2 + 截图
   └─ 方法3 + 注意事项

3. 结尾（150字）
   ├─ 互动问题
   └─ 引导关注
```

## 标题公式

### 反差型
> "别只做X，这才是Y"
> "X没用，试试这个Y"

**示例**：
- "别只学Prompt，这才是用AI的关键"
- "AI写作没用？试试这套方法"

### 价值型
> "如何用X实现Y？（附方法）"
> "我用X做了Y，效果超出预期"

**示例**：
- "如何用Claude Code提升3倍开发效率？（附实战）"
- "我用AI写公众号，阅读量翻倍了"

### 角色型
> "X不仅是Y，更是你的Z"
> "别把X当Y，它是Z"

**示例**：
- "Claude Code不仅是编程助手，更是你的思维伙伴"
- "别把AI当作者，它是你的编辑"

### 实战型
> "我用X做了Y（附教程）"
> "X天搞定Y，我的实战经验"

**示例**：
- "我用AI写了100篇文章，总结出3个规律"
- "7天搞定公众号爆文，我的实战经验"

## 写作节奏控制

### 句子长度
- **最佳**：15-25字
- **上限**：不超过30字
- **变化**：长短结合，有节奏感

**示例**：
> "Claude Code出了。（7字）
> 我用了两周。（7字）
> 确实比Cursor好用。（10字）
> 快了2倍，准了30%。（10字）
> 我测下来的数据是这样的。（12字）"

### 段落长度
- **手机屏显示**：3-5行
- **一个段落**：只讲一个观点
- **段落之间**：空行分隔

### 标点使用
- **多用句号**：句子短，有力量
- **少用逗号**：避免长句
- **慎用感叹号**：显得不专业

**示例**：
```
✅ Claude Code出了。快。准。好用。

❌ Claude Code终于发布了，不仅速度非常快，而且准确度极高，真的超级好用！！！
```
